Auth con Google + Sesiones

Cómo funciona el flujo OAuth 2.0 paso a paso:

1
Usuario hace clic en "Continuar con Google"
El servidor redirige a accounts.google.com con tu client_id y los scopes que pediste.
2
Google autentica al usuario
Si acepta, Google redirige de vuelta a tu /auth/google/callback con un code temporal.
3
Tu servidor intercambia el code por tokens
Passport hace esta solicitud automáticamente y recibe el perfil del usuario.
4
Sesión creada en el servidor
express-session guarda los datos y envía una cookie connect.sid al navegador.
5
Frontend guarda en localStorage
El dashboard llama a /api/me y guarda el perfil en localStorage para acceso rápido offline.
